# 인터뷰 준비 (Interview Prep)

페르소나 기반으로 사용자 인터뷰 스크립트를 생성합니다.
The Mom Test 원칙을 적용하여 편향 없는 질문을 구성합니다.

## 인수

- `$ARGUMENTS`: overview.md 경로 또는 프로젝트 자연어 설명

## 프로세스

### 1. 맥락 파악

`$ARGUMENTS`가 파일 경로이면 해당 파일을 읽고, 아닌 경우 자연어 설명에서 추출:
- 프로젝트 목적
- 타겟 사용자 / 페르소나
- 핵심 가정 (있는 경우)

### 2. The Mom Test 원칙 적용

**금지 질문 (Leading Questions):**
- "이 기능이 좋으시겠죠?" → 예/아니오 유도
- "이런 앱을 사용하시겠어요?" → 미래 행동 예측 불가
- "이 문제가 심각하지 않나요?" → 감정 유도

**좋은 질문 (The Mom Test):**
- "마지막으로 [문제 상황]을 겪었을 때 어떻게 해결하셨나요?"
- "현재 [행동]을 할 때 가장 귀찮은 점은 뭔가요?"
- "그 문제를 해결하기 위해 돈이나 시간을 써본 적 있나요?"

### 3. 인터뷰 스크립트 생성

페르소나별로 다음 구조의 스크립트를 생성합니다:

```markdown
# 인터뷰 스크립트: [페르소나 이름]

## 인터뷰 목적
[검증하려는 가정 1~3개]

## 오프닝 (2분)
- 자기소개 + 인터뷰 목적 간략 설명
- "정답은 없습니다. 솔직한 경험을 듣고 싶습니다"

## 현재 행동 탐색 (10분)
1. [구체적 질문 — 과거 행동 기반]
2. [구체적 질문 — 빈도/맥락]
3. [구체적 질문 — 현재 해결 방법]

## 문제 깊이 탐색 (10분)
4. [구체적 질문 — 페인포인트 심층]
5. [구체적 질문 — 시간/비용 투자]
6. [구체적 질문 — 포기한 시도]

## 가치 검증 (5분)
7. [구체적 질문 — 이상적 해결책 묘사]
8. [구체적 질문 — 지불 의사]

## 클로징 (3분)
- "다른 이야기하고 싶은 것이 있나요?"
- 다음 인터뷰이 소개 요청

## 메모 가이드
- [ ] 감정적 반응 기록 (말투, 표정)
- [ ] 구체적 수치 기록 (빈도, 금액, 시간)
- [ ] "흥미로운 발견" 별도 표시
```

### 4. 출력

- 페르소나별 인터뷰 스크립트 파일 생성 (`docs/interview-[persona].md`)
- 전체 가정 목록과 각 질문이 검증하는 가정 매핑

## full-auto와의 관계

이 명령어는 **standalone**입니다. full-auto 워크플로우에 내장되지 않습니다.
Phase 0 전에 사용자가 선택적으로 실행하여 인터뷰를 준비하고,
인터뷰 결과를 `/interview-summary`로 분석한 후 full-auto에 반영할 수 있습니다.
