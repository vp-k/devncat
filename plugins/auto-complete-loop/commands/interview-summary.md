# 인터뷰 요약 (Interview Summary)

사용자가 제공한 인터뷰 기록을 분석하여 패턴을 추출하고 요구사항을 도출합니다.

## 인수

- `$ARGUMENTS`: 인터뷰 기록 파일 경로 (복수 가능) 또는 직접 텍스트

## 프로세스

### 1. 기록 로드

`$ARGUMENTS`에서 인터뷰 기록을 로드합니다:
- 파일 경로이면 Read로 읽기
- 텍스트이면 그대로 사용
- 복수 파일이면 모두 로드

### 2. 패턴 추출

인터뷰 기록에서 다음을 추출합니다:

**행동 패턴:**
- 공통된 현재 행동/워크플로우
- 자주 언급된 불편/페인포인트
- 기존 해결 방법 (경쟁 제품, 우회 방법)

**감정 신호:**
- 강한 감정적 반응 (좌절, 흥분, 체념)
- "꼭 필요하다" vs "있으면 좋겠다" 구분

**수치 데이터:**
- 빈도 (얼마나 자주 겪는가)
- 비용 (시간/돈을 얼마나 쓰는가)
- 규모 (몇 명이 영향 받는가)

### 3. 가정 검증 결과

`docs/overview.md`가 있으면 기존 가정과 대조:

```markdown
| 가정 | 검증 결과 | 근거 |
|------|----------|------|
| [가정 1] | 확인/반박/불충분 | [인터뷰이 발언 인용] |
```

### 4. 인사이트 + 요구사항 도출

```markdown
## 핵심 인사이트
1. [인사이트] — 근거: [인터뷰이 N명 공통]
2. ...

## 도출된 요구사항
- [기존 기능 강화 또는 새 기능]
- 우선순위: [MoSCoW 분류]

## 수정 권장사항
- overview.md 업데이트 필요 항목
- 페르소나 보정 필요 항목
- 새로 발견된 Non-Goals
```

### 5. 출력

- `docs/interview-insights.md` 생성
- overview.md 수정이 필요한 경우 구체적 수정안 제시 (직접 수정은 사용자 확인 후)

## full-auto와의 관계

이 명령어는 **standalone**입니다.
인터뷰 결과를 overview.md에 반영한 후 full-auto를 실행하면,
Phase 0에서 검증된 가정 기반으로 더 정확한 기획을 수행합니다.
