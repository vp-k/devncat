# Post-Analysis: Launch Analysis

기획 문서와 코드를 기반으로 런치 전략 초안을 생성합니다.
AI가 코드/문서에서 추출 가능한 부분은 자동 작성하고, 외부 조사가 필요한 부분은 `[조사 필요]`로 표시합니다.

## 입력

- 기획 문서 (docs/ 내 overview.md, SPEC.md, personas 등)
- 소스 코드 (pubspec.yaml, package.json 등 프로젝트 메타데이터)
- 오케스트레이터가 수집한 공통 맥락

## 분석 절차

### Step 1: 포지셔닝 메시지 초안

기획 문서에서 추출:

1. **Problem Statement**: overview.md 또는 SPEC.md에서 문제 정의 추출
2. **Solution Statement**: 핵심 기능 3개 이내로 요약
3. **Elevator Pitch**: 한 문장 (Problem → Solution → Benefit 구조)
4. **Tagline 후보**: 3개 제안

문서에 명시적 Problem Statement가 없으면 코드의 핵심 기능에서 역추론합니다.

### Step 2: 타겟 사용자 특성

기획 문서의 페르소나에서 추출:

1. **Primary Persona**: 이름, 특성, 핵심 니즈
2. **Secondary Persona**: (있을 경우)
3. **사용자 여정 요약**: 진입 → 핵심 가치 발견 → 습관화

페르소나 문서가 없으면 코드의 기능 구조에서 추론:
- 어떤 문제를 해결하는 기능인지
- 복잡도 수준 (초보자/전문가 대상)
- 사용 빈도 패턴 (일상/비정기)

### Step 3: 채널 제안 프레임

> 이 섹션은 외부 조사가 필요합니다.

프레임만 제공하고 구체적 내용은 `[조사 필요]`:

| 채널 유형 | 후보 | 적합 이유 | 예상 비용 | 우선순위 |
|----------|------|----------|----------|---------|
| 오가닉 | [조사 필요] | ... | [조사 필요] | [조사 필요] |
| 페이드 | [조사 필요] | ... | [조사 필요] | [조사 필요] |
| 커뮤니티 | [조사 필요] | ... | [조사 필요] | [조사 필요] |
| PR/미디어 | [조사 필요] | ... | [조사 필요] | [조사 필요] |

### Step 4: 런치 타임라인 초안

> 이 섹션은 외부 조사가 필요합니다.

일반적인 런치 프레임워크 기반 초안:

| 단계 | 기간 | 주요 활동 | 상세 |
|------|------|----------|------|
| Pre-launch | [조사 필요] | 랜딩페이지, 웨이트리스트, 티저 | [조사 필요] |
| Soft Launch | [조사 필요] | 베타 테스터, 피드백 수집 | [조사 필요] |
| Public Launch | [조사 필요] | 채널별 동시 런치, PR | [조사 필요] |
| Post-launch | [조사 필요] | 데이터 분석, 반복 개선 | [조사 필요] |

### Step 5: 앱스토어 메타데이터 초안 (Flutter/모바일 한정)

`pubspec.yaml` 또는 모바일 프로젝트 설정에서 앱 정보를 추출합니다.
웹 전용 프로젝트이면 이 Step을 스킵합니다.

1. **앱 이름**: pubspec.yaml의 name/description에서 추출
2. **짧은 설명** (80자): Elevator Pitch 기반
3. **긴 설명** (4000자): 기능 목록 + 혜택 중심
4. **키워드**: 기능에서 추출한 검색 키워드 10개
5. **카테고리**: 앱 기능에 맞는 스토어 카테고리 추천
6. **스크린샷 가이드**: 핵심 화면 5개 + 각 화면의 캡션 제안
